## Rapport d’exécution SCRUM / GitHub Projects

- **Cadence** : Sprints de 2 semaines, réunions quotidiennes (15 min), review & rétro chaque sprint.
- **Outils** : GitHub Projects (Vue Kanban), champs personnalisés (Story Points, Sprint).
- **Metrics** :
  - Burndown automatique basé sur champs points.
  - Lead time moyen visé < 4 jours.
  - Taux de couverture tests > 70 %.
- **Processus** :
  1. Création issues depuis backlog, assignation à un sprint.
  2. PR obligatoire avec relecteur, checks CI obligatoires.
  3. Tag `documentation` pour artefacts, `infrastructure` pour scripts.
- **Communication** : compte-rendu hebdo envoyé au sponsor (Evidian/Maidis).
